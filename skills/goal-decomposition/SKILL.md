# Goal Decomposition

Goal: turn one big, vague ambition into a shape the Growth tree can hold —
a small number of branches, each with a first concrete leaf — without
flattening the excitement out of it.

## Procedure

1. **Restate the goal** in their words and ask the one clarifying question
   that most changes the decomposition (usually: what does "done" look like,
   or what's the deadline/stakes?). Skip this if the conversation already
   answered it.
2. **Find the 2–4 natural branches** — the independent workstreams inside the
   goal. Test each: could they make progress on it while ignoring the others?
   If not, it's a step, not a branch.
3. **For each branch, name the first leaf**: the smallest action that is
   genuinely startable this week, phrased as something they do, not something
   that happens.
4. **Suggest a review cadence** matched to the goal's tempo (weekly for
   active pushes, monthly for slow burns) — as a habit, not a lecture.
5. **Offer to put it in the tree**: propose the structure through the normal
   tree-proposal mechanism, one proposal at a time, starting with the piece
   they reacted to most. All the standing tree rules apply — confidence gate,
   real node ids, one proposal per message, their approval before anything
   is created.

## Boundaries

- 2–4 branches, no more. A goal that needs six branches is probably two
  goals — say so.
- Don't decompose enthusiasm into homework. If they're still dreaming out
  loud, reflect and explore; decompose when they're ready to move.
- Never create tree structure silently or describe the proposal syntax to
  them — the proposal blocks handle it.
